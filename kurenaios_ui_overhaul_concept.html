Breadth-first traversal

How a queue turns a graph into layers, and why that matters when the shortest unweighted path is the real question.

Core idea

A graph is made from vertices joined by edges. Breadth-first traversal explores every neighbour of a starting vertex before moving outward to the next layer. The algorithm therefore visits vertices in increasing distance from its source.

The queue is load-bearing. Each newly discovered vertex is placed at the rear, while the vertex at the front is removed and explored next. Because a queue is first in, first out, earlier layers are exhausted before later ones begin.

要
Exam lens

State the data structure and the marking step. Saying only “visit every node” misses why the traversal terminates and how repeated visits are prevented.

Reference implementation

The set named seen records a vertex when it enters the queue. Marking it at that moment, rather than when it leaves, prevents the same vertex being queued by multiple neighbours.

breadth_first.py 
from collections import deque

def breadth_first(graph, start):
    queue = deque([start])
    seen = {start}
    order = []

    while queue:
        vertex = queue.popleft()
        order.append(vertex)

        for neighbour in graph[vertex]:
            if neighbour not in seen:
                seen.add(neighbour)   # mark before queuing
                queue.append(neighbour)

    return order

Complexity and use

With an adjacency-list representation, every vertex is processed once and every edge is examined once, giving time complexity O(V + E). The queue and visited set require up to O(V) additional space.

Breadth-first traversal is the right choice for finding the fewest-edge route through an unweighted graph. For weighted routes, the order must account for cost; that is where Dijkstra’s algorithm takes over.
