Binary Trees & Traversal

Paper 4 · Practical Mastery 68% Last opened 2 days ago Pass the quiz → +40 XP

A binary tree is a hierarchical data structure in which every node holds a value and at most two references — conventionally Left and Right. The topmost node is the root; nodes with no children are leaves. Unlike the linear structures you met in Topic 18, a tree encodes an ordering spatially, which is what makes logarithmic search possible.

A binary search tree (BST) adds one rule on top: for any node, everything in its left subtree is smaller, and everything in its right subtree is larger. That single invariant is doing all the work — every operation you'll write this chapter is just a restatement of it.

豆
Mame chishiki — worth remembering

An in-order traversal of a valid BST always visits values in ascending order. Examiners love this: if a trace question yields an unsorted sequence, you've mis-read the tree — go back before you lose easy marks.

§ 01Insertion

Insertion follows one comparison per level: smaller goes left, larger goes right, and a null reference marks the vacancy where the new node belongs. Written recursively, the code almost reads as the definition itself:

BinarySearchTree.cs C# ⧉ copy 
1public Node Insert(Node root, int value)
2{
3    // A null slot is exactly where the value belongs.
4    if (root is null) return new Node(value);
5
6    if (value < root.Value)
7        root.Left  = Insert(root.Left, value);
8    else
9        root.Right = Insert(root.Right, value);
10
11    return root; // ancestors are unchanged
12}

§ 02The three traversal orders

Each traversal is defined by where the visit happens relative to the two recursive calls. Fix that in your head and you never need to memorise the outputs:

Pre-order visit → L → R

Root first. Used to serialise or copy a tree — you must know the parent before you can attach children.

In-order L → visit → R

Left first. On a BST this yields sorted output — the exam's favourite property.

Post-order L → R → visit

Root last. Used to delete or free a tree safely — children go before their parent.

For the 9618 paper you should be able to trace all three by hand on a tree of eight to ten nodes, and to write the in-order version in pseudocode from memory. The recursive skeleton is identical in all three cases — only one line moves.
